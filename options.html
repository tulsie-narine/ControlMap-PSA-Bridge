ScalePad Platform API

Integrations

Compliance-data integrations surface in the on-page overlay panel. Click a tile to configure. Each integration is a folder in integrations/ — see docs/BUILDING_INTEGRATIONS.md to build your own.

PSA

Click a tile to configure credentials and ticket routing. The active PSA is where Create Ticket sends tickets.

Ticket defaults

Loaded from the selected PSA after a successful connection test. Pre-selected in the Create Ticket modal; changeable per ticket.

Tenant mappings (ControlMap → client → PSA company)

Map a ControlMap tenant subdomain (e.g. carb1 from carb1.app.ctrlmap.com) to the ScalePad client, and optionally pre-pick the PSA company. Manual mappings take priority over auto-detection.

Distributors

Map each Quoter Supplier name to an order email and, optionally, a distributor API so Atlas can push the PO directly. The name must match the supplier shown on Quoter items (e.g. Ingram Micro, TD SYNNEX, D&H). Credentials are stored locally in your browser only and are never sent to the page.

Configure PSA

Create an API-only user in Autotask (Admin → Resources → API User). Zone is auto-detected.
Create API Member keys in ConnectWise (System → Members → API Members). The clientId is registered free at developer.connectwise.com. Cloud sites: api-na / api-eu / api-au .myconnectwise.net.
In HaloPSA: Configuration → Integrations → HaloPSA API → Applications → add an app with "Client ID and Secret (Services)" auth and ticket read/edit + client read scopes.
